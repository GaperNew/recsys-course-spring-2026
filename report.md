# Отчет по домашнему заданию 2

## Метод

В контрольной группе используется исходный рекомендатель SasRec-I2I. В экспериментальной группе используется адаптивный гибридный рекомендатель, который объединяет несколько источников кандидатов: персональные рекомендации HSTU, item-to-item рекомендации SasRec и item-to-item рекомендации LightFM.

Решение построено как схема candidate generation + reranking. На первом этапе рекомендатель собирает кандидатов из пользовательского списка HSTU и из соседей последних треков текущей сессии. На втором этапе кандидаты ранжируются по скору, который учитывает позицию рекомендации в исходной модели, время прослушивания anchor-трека и свежесть этого трека внутри сессии.

Дополнительно используется легкая online-адаптация под пользователя. Для каждого источника рекомендаций хранится сглаженная оценка качества. Если рекомендация из некоторого источника приводит к более долгому прослушиванию, вес этого источника для данного пользователя повышается. Если пользователь быстро пропускает трек, источник получает меньший вес в следующих шагах сессии.

## Консервативный fallback

Так как SasRec-I2I является сильным базовым решением, treatment не заменяет его безусловно. Рекомендация гибридной модели используется только в тех случаях, когда предыдущий трек был прослушан достаточно долго, а скор гибридного кандидата выше скора baseline-кандидата с заданным запасом. В остальных случаях возвращается рекомендация SasRec-I2I.

Такой подход снижает риск ухудшения качества относительно control-группы и позволяет использовать гибридную модель только в ситуациях, где она имеет достаточно уверенности.

## Результат

На локальном A/B-тесте с 3000 эпизодами получен статистически значимый прирост по целевой метрике mean_time_per_session:

- control: 7.2684
- treatment: 7.6516
- effect: +5.27%
- confidence interval: от +1.08% до +9.46%
- significant: true

Также наблюдается рост среднего числа треков в сессии:

- mean_tracks_per_session: +3.02%
- significant: true

Таким образом, предложенный адаптивный гибридный рекомендатель улучшает среднее время сессии по сравнению с исходным SasRec-I2I baseline.
